## 學到的經驗
- **Characterization test 的價值**: getNearby 名稱暗示「過濾 radius 內實體」,實際是 cell 範圍查詢(不過濾實際距離)。測試把這個非直觀行為記錄下來,避免日後誤用。
- **既有純函式補測試的 TDD 節奏**: 一次寫一批比「嚴格一個測試 → 看紅 → 看綠」更實際,因為實作正確時測試直接綠燈。但擴充新方法仍應嚴守紅綠燈。
- **TDD 技能明文允許「既有碼補測試」**,但擴充新行為必須遵守「NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST」鐵律。
